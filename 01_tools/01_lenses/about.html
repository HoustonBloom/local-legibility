About

A vault-agnostic publishing system.

Turn any folder of markdown into curated HTML intelligence packets and a full local-first preview website. Bring your own folder structure. The packet adapts.

Two subsystems

One design system, two ways to publish.

Pipeline

Content Publishing Pipeline

Renders curated HTML frames from selected files, shaped by voice and audience.

  • Cut = config (selection + voice + output target)
  • Frame = rendered HTML output of one cut
  • Window = multiple frames with tabs in one file
  • Narrative = reusable voice preset (8 ship with the packet)
  • Lens = layout pattern (7 specs, 6 live examples)
  • Content-tag = visibility array string sourcing a cut

Requirements

What the packet needs from your folder.

Required: markdown files.

Files with .md extension and optional YAML frontmatter. For the preview site, add a visibility field to any file you want published.

Recommended: top-level folders that group content by kind.

The reference uses Initiatives, Products, Research, Content/Published. A type field in frontmatter helps distinguish kinds of content.

Optional: a daily changelog and session-carryover file.

Powers the timeline activity view and current-work display. Not required for frames.

Frontmatter is optional for the pipeline.

Most lenses can work from filenames, headings, and content alone. The packet is forgiving by design.

Design principles

What this system encodes.

Four commitments that govern every decision in the packet. Every lens, atmosphere, skill, and convention is scored against these. They are also the tagging vocabulary for Activity Log entries (see the Workflow tab).

Make harm legible.

You cannot organize against what you cannot see. Every frame is inspectable HTML. No black-box rendering. Restore the feedback loop.

Provide an exit.

Content stays as portable markdown. No lock-in to the packet, the AI, or the cloud. You can leave at any time with your folder intact.

Design for sufficiency.

The pipeline is one script. The design system is one stylesheet's worth of tokens. Nothing is optimized for scale it doesn't need.

Scale out, not up.

Federation over merger. Sovereign cells connecting laterally, sharing protocols not ownership. The packet installs anywhere, reports to no one.
